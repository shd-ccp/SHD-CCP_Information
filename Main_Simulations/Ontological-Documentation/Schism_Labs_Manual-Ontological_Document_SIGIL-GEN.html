System Manual

The Ontological Source Tracer is a specialized instrument for mapping knowledge topography. Unlike linear file systems that organize data by name or date, this system organizes data by genesis lineage and geometric perspective.

Public Test Build (Forward Facing)

This is the forward-facing public test version hosted via the SHD-CCP GitHub. In this early iteration, users maintain their own JSON data structure locally. As the system matures, cloud storage and direct AI integration features will be enabled.

Research & Development: The core development is led by Schism Labs, operators of the Genesis Chain. The physics, mathematics, and geometric semiotics systems are provided by BioChain AI.

We have open-sourced the 64-bit SHD-CCP (Symbolic High-Dimensional Context Compression Protocol) to enable universal AI interoperability. This protocol is optimized for growing general intelligence through non-integer based reasoning, moving beyond the limitations of traditional linear linguistic models.

2. The SHD-CCP Protocol

The SHD-CCP (Slice-Harmonic-Depth Complex-Coordinate-Plane) is the underlying data structure that enables the visual and cryptographic functions of the tracer.

The 64-Bit "Sandwich" Layout

We do not store data as arbitrary strings. We pack metadata into a precise 64-bit Hex structure, optimized for both 3D rendering and AI interpretation.

HEX ID STRUCTURE: [ 00 11 22 33 44 55 66 77 ] (16 Chars)

ROW 0 [0-1]: STRUCTURE ID (Semiotics) -> Defines the geometric primitive (Square, Circle, etc.)
ROW 1 [2-3]: SCALE A (Buffer) -> Defines expansion radius
ROWS 2-5 [4-11]: QUATERNION (Math) -> Defines vector orientation (x,y,z,w)
ROW 6 [12-13]: SCALE B (Buffer) -> Defines harmonic resonance
ROW 7 [14-15]: DYNAMICS (Physics) -> Defines frequency/color (Temporal state)

Geometric Translation

The system reads this hex string and projects it into a 4x4x4 Voxel Cube. By taking cross-sections (Slices) of this cube, we generate the 2D Sigil.

  • Outer Square: Represents the Root/Course Data.
  • Middle Circle: Represents the Class/Branch Data.
  • Inner Triangle: Represents the Hash/File Data.

3. Hyperbolic Space & AI Translation

This system is designed to bridge the gap between human readability and machine comprehension.

AI-to-AI Translation

Large Language Models (LLMs) struggle with raw images but excel with tokens. The SHD-CCP protocol converts complex geometric relationships into a 16-character string.

The Bridge

When one AI passes a document to another, it doesn't need to describe the file. It passes the SHD-CCP Packet (e.g., A4F9...). The receiving AI decodes the "Sandwich" to instantly understand:

  1. Which Project (Genesis Seed) it belongs to.
  2. What perspective (Branch) is required to view it.
  3. What state (Dynamics) the document is in.

Advanced Reasoning Capabilities

The 3-Nested Minimum Data Protocols do more than allow instant lookup across multiple perspectives. They enable inverse deductive reasoning and associative reasoning to occur within the machine logic.

By structuring data geometrically rather than linearly, the system facilitates the creation of higher "context-aware" perspectives. This allows for multi-system inference across an almost infinite combination of linear parameterized training systems, effectively allowing the AI to "think" in shapes and relations rather than just sequences of words.

Hyperbolic Visualization

The "Global Chain" viewer renders the database not as a list, but as a constellation. The connections between nodes (Files) and hubs (Folders) represent Semantic Distance rather than just alphabetical order. This allows the user (and the AI) to visualize the "weight" and "gravity" of different knowledge clusters.

4. Chain Sovereignty

The Ontological Source Tracer is a Local-First application. It creates a verified chain of custody without relying on external servers.

The Local Chain

All data is stored in a JSON structure within the browser's LocalStorage. This JSON file *is* your database. It contains:

  • The entire directory tree.
  • The cryptographic seeds for every folder.
  • The evolution logs for every file.

Portability

By using the Export function, you create a portable "Snapshot" of your entire knowledge base. This file can be:

  • Sent to another user.
  • Backed up to a hard drive.
  • Injected into a fresh instance of the Tracer to instantly restore the full environment.

5. Step-by-Step Workflow

Step A: Initialization

1. Open the sidebar and click Module (Folder).
2. Name your project (e.g., "Project Alpha").
3. The system mints a Genesis Seed automatically.

Step B: Derivation

1. Enter the new Module folder.
2. Click Doc Source (File).
3. Select the Branch that matches your content's intent (e.g., "FRONT" for an intro, "RIGHT" for a draft).
4. The system rotates the Genesis Seed to create a unique File ID.

Step C: Evolution

1. Select the file to open the Inspector.
2. Use the Progress Log to add entries.
3. Selecting a different "Spin State" (e.g., rotating from Front to Back) updates the voxel visualization to reflect the document's maturity.

Step D: Authentication

1. Click Download PNG in the inspector.
2. The resulting image contains the visual Sigil AND the embedded Hex ID (Steganography).
3. Use the Scanner Tab to re-import this image later, instantly locating the source record in your database.

6. System Administration

The Admin Console provides low-level access to the data structure.

Commands

  • stats: View total node count and database size.
  • export: Trigger a raw JSON dump to the clipboard.
  • nuke: Emergency wipe of all local data (Reset to Factory).

JSON Injection

The "Raw JSON Injection" bay allows you to manually patch the database. You can paste a previously exported JSON block here to merge or overwrite the current state. This is useful for syncing data between different team members manually.
